编译选项

在编写 ts 的过程中，每次修改代码都要重新使用 tsc 编译

可以使用 -w 参数“监视文件”

有时间间隔

使用 tsc 编译所有文件

在目录下新建 jsconfig.json文件

配置选项

  1. “include” 用来指定哪些 ts 文件需要被编译

  2. “exclude” 表示不包含的文件

  3. “compilerOptions”重要的一些配置

    • “target” 指定 ts 被编译的版本

    • “module” 指定模块化版本

    • “lib” 指定项目中要使用的库

    • “outDir” 指定编译后文件所在的目录

    • “outFire” 编译为一个文件

    • “allowJs” 是否对 js 文件进行编译 默认 false

    • “checkJs” 是否对 js 文件进行检查 默认 false

    • “removeComments” 是否移除注释 默认 false

    • “noEmit” 不生成编译后的文件 默认 false

    • “noEmitOnError” 代码中有错误不生成编译后的文件 默认 false

    • “alwaysStrict” 编译后的文件是否使用严格模式 默认 false

    • “noImplicitAny” 不允许隐式的 any

    • “noImplicitThis” 不允许隐式的 this

    • “strict” 所有检查的开关
